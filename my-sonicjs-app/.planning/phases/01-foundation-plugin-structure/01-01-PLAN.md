  <name>Task 3: Create SQL migration for redirects tables</name>
  <files>
    migrations/032_redirect_plugin.sql
  </files>
  <action>
Create the database migration for the redirect management plugin.

**Plugin registration:**
INSERT INTO plugins with id='redirect-management', category='utilities', status='inactive', default settings JSON.
Use ON CONFLICT(id) DO UPDATE for idempotency.

**Redirects table:**
CREATE TABLE IF NOT EXISTS redirects with:
- id TEXT PRIMARY KEY
- source TEXT NOT NULL (URL pattern)
- destination TEXT NOT NULL (target URL)
- match_type INTEGER NOT NULL DEFAULT 0 (0=exact, 1=partial, 2=regex)
- status_code INTEGER NOT NULL DEFAULT 301
- is_active INTEGER NOT NULL DEFAULT 1
- created_by TEXT NOT NULL REFERENCES users(id)
- created_at INTEGER NOT NULL (milliseconds)
- updated_at INTEGER NOT NULL (milliseconds)

**Indexes:**
- idx_redirects_source ON redirects(source) - primary lookup
- idx_redirects_active ON redirects(is_active) - filter active redirects
- idx_redirects_match_type ON redirects(match_type) - filter by type

**Analytics table:**
CREATE TABLE IF NOT EXISTS redirect_analytics with:
- id TEXT PRIMARY KEY
- redirect_id TEXT NOT NULL REFERENCES redirects(id) ON DELETE CASCADE
- hit_count INTEGER NOT NULL DEFAULT 0
- last_hit_at INTEGER (nullable, milliseconds)
- created_at INTEGER NOT NULL
- updated_at INTEGER NOT NULL

Index: idx_redirect_analytics_redirect_id

**Sample data:**
INSERT OR IGNORE sample redirects demonstrating different configurations:
1. /old-page -> /new-page, exact match, 301, active
2. /blog/old-post -> /blog/new-post, exact match, 302, active
3. /temp-page -> /permanent-page, exact match, 307, inactive (testing disabled state)
4. /gone-page -> (empty or same), exact match, 410, active (gone page example)

Use strftime('%s', 'now') * 1000 for timestamps.
Use 'admin-user-id' as placeholder for created_by (or query for actual admin user).
