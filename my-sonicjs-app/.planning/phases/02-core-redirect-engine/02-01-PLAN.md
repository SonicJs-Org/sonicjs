  <name>Task 1: Create URL normalizer utilities</name>
  <files>src/plugins/redirect-management/utils/url-normalizer.ts</files>
  <action>
Create URL normalization module with two exported functions:

1. `normalizeUrl(url: string): string`
   - Convert to lowercase for case-insensitive matching
   - Remove trailing slash EXCEPT for root "/"
   - Handle edge cases: empty string returns "/", null/undefined returns "/"
   - Do NOT decode URI components (preserve encoded characters for matching)

2. `normalizeUrlWithQuery(url: string, includeQuery: boolean): string`
   - Call normalizeUrl() first
   - If includeQuery is false, strip query string (everything after ?)
   - If includeQuery is true, keep query string intact
   - Return normalized URL

Example transformations:
- "/Blog" -> "/blog"
- "/page/" -> "/page"
- "/" -> "/"
- "/page?ref=email" with includeQuery=false -> "/page"
- "/page?ref=email" with includeQuery=true -> "/page?ref=email"

Use standard string operations. Do NOT use external libraries like normalize-url (per research recommendation to keep it simple for now).
  </action>
