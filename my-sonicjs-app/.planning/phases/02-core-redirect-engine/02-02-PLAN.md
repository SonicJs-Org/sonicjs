Create validation module with circular detection and URL validation.

1. `ValidationResult` interface:
```typescript
interface ValidationResult {
  isValid: boolean
  error?: string       // Error message if invalid
  warning?: string     // Warning message (valid but concerning)
  chainLength?: number // Number of hops in redirect chain
  chainUrls?: string[] // URLs in the chain
}
```

2. `detectCircularRedirect(source: string, destination: string, existingRedirects: Map<string, string>): ValidationResult`
   - Detect if adding source->destination creates a cycle
   - existingRedirects is a Map from source URL to destination URL
   - Algorithm (visited-set pattern from research):
     a. Create visited Set with source (normalized lowercase)
     b. Start at destination, normalize it
     c. While current exists in existingRedirects:
        - If current is in visited: CIRCULAR - return invalid
        - Add current to visited, add to chain
        - Move to next destination
        - If chain length > 10: break with error (safety limit)
     d. If chain length >= 3: return valid but with WARNING
     e. Return valid
   - Return chainUrls for debugging (source, dest1, dest2, ...)

3. `validateUrl(url: string): ValidationResult`
   - Check URL is non-empty
   - Check URL starts with "/" (relative) OR starts with "http://" or "https://" (absolute)
   - Do NOT validate if destination exists (that's a separate concern)
   - Basic format check only

4. `validateRedirect(source: string, destination: string, existingRedirects: Map<string, string>): ValidationResult`
   - Convenience function that runs both validations
   - First validate source URL format
   - Then validate destination URL format
   - Finally check for circular redirects
   - Return first error, or warning from circular detection, or valid

Export all three functions and ValidationResult interface.

Implementation note: Normalize URLs to lowercase before comparison (use simple `.toLowerCase()` - don't import url-normalizer to avoid circular deps).
