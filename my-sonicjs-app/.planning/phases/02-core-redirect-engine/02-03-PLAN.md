Add these methods to RedirectService class:

1. `async create(input: CreateRedirectInput, userId: string): Promise<RedirectOperationResult>`
   - Generate UUID for id: `crypto.randomUUID()`
   - Set defaults: matchType=0, statusCode=301, isActive=true, includeQueryParams=false, preserveQueryParams=false
   - Load existing redirects for validation: `await this.getAllSourceDestinationMap()`
   - Call validateRedirect(source, destination, existingMap)
   - If validation fails: return { success: false, error: validation.error }
   - Normalize source URL before storage (lowercase, no trailing slash)
   - INSERT into redirects table
   - Return { success: true, redirect: newRedirect, warning: validation.warning }

2. `async getById(id: string): Promise<Redirect | null>`
   - SELECT from redirects WHERE id = ?
   - Map database row to Redirect type
   - Return null if not found

3. `async update(id: string, input: UpdateRedirectInput): Promise<RedirectOperationResult>`
   - Fetch existing redirect by ID
   - If not found: return { success: false, error: 'Redirect not found' }
   - If source or destination changed: validate again
     - Build map excluding current redirect
     - validateRedirect(newSource, newDestination, mapWithoutSelf)
   - UPDATE only provided fields, set updated_at = Date.now()
   - Return updated redirect

4. `async delete(id: string): Promise<RedirectOperationResult>`
   - DELETE FROM redirects WHERE id = ?
   - Return { success: true } if rowCount > 0
   - Return { success: false, error: 'Redirect not found' } otherwise

5. `async list(filter?: RedirectFilter): Promise<Redirect[]>`
   - Build SELECT with optional WHERE clauses
   - Filter by isActive, statusCode, matchType if provided
   - LIKE search on source and destination if search provided
   - Apply LIMIT (default 50) and OFFSET
   - ORDER BY created_at DESC
   - Map rows to Redirect[]

6. `async getAllSourceDestinationMap(): Promise<Map<string, string>>`
   - Helper method for circular detection
   - SELECT source, destination FROM redirects WHERE is_active = 1
   - Return Map<normalizedSource, destination>

7. `async count(filter?: RedirectFilter): Promise<number>`
   - COUNT(*) with same filters as list
   - Used for pagination

8. `async lookupBySource(normalizedSource: string): Promise<Redirect | null>`
   - Used by middleware to find redirect by source URL
   - SELECT from redirects WHERE LOWER(source) = ? AND is_active = 1 LIMIT 1
   - Map database row to Redirect type
   - Return null if not found
