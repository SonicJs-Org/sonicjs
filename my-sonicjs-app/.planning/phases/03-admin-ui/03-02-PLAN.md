  <name>Task 2: Add form route handlers</name>

  <action>
Add route handlers for create and edit forms to the existing admin routes file:

1. Import renderRedirectFormPage from '../templates/redirect-form.template'
2. Import CreateRedirectInput, UpdateRedirectInput from '../types'

3. GET /admin/redirects/new (create form page):
   - Extract ref query param (referrer params)
   - Render form with isEdit: false, referrerParams: ref

4. GET /admin/redirects/:id/edit (edit form page):
   - Extract id from params
   - Extract ref query param
   - Fetch redirect via service.getById(id)
   - If not found: return 404 or redirect to list with error
   - Render form with isEdit: true, redirect: result, referrerParams: ref

5. POST /admin/redirects (create redirect):
   - Parse form body (source, destination, status_code, match_type, include_query_params, preserve_query_params, active)
   - Convert checkbox values: 'on' or '1' -> true, missing -> false
   - Convert numeric values: status_code and match_type to numbers
   - Build CreateRedirectInput
   - Get userId from context (c.get('user')?.id or 'system' if not available)
   - Call service.create(input, userId)
   - If success: redirect to /admin/redirects with success message (via query param or flash)
   - If error: re-render form with error message

6. PUT /admin/redirects/:id (update redirect):
   - Parse form body (same fields as POST)
   - Build UpdateRedirectInput (only include fields that are present)
   - Call service.update(id, input)
   - If success: redirect to /admin/redirects
   - If error: re-render form with error message and existing values

7. Form data parsing helper:
   - parseFormData(body): Extract and convert form values
   - Handle checkbox conversion (checkbox sends 'on' when checked, nothing when unchecked)
   - Use parseInt for numeric fields with defaults
