Add route handlers for single and bulk delete operations:

   - Extract id from params
   - Call service.delete(id)
   - If success: return 200 with success message (HTMX will handle reload)
   - If error (not found): return 404 with error message
   - Response format: JSON { success: true } or { success: false, error: string }

   - Collect results (count successes and failures)
   - Return summary: { success: true, deleted: N, failed: M }

   - Return user-friendly error messages

4. HTMX response considerations:
   - DELETE returns minimal response (HTMX handles page reload via client-side)
   - Set HX-Trigger header if you want to trigger events
   - Alternative: Return HX-Redirect header to trigger navigation

Note: The list template's HTMX call will reload the page after successful delete. The route just needs to return success/error status.
