  <name>Task 1: Create REST API route handlers</name>
  <files>src/plugins/redirect-management/routes/api.ts</files>
  <action>
Create a new file `src/plugins/redirect-management/routes/api.ts` with RESTful JSON API endpoints.

Follow the complete example from 05-RESEARCH.md. Key implementation details:

1. **Create Hono router with apiError helper:**
   - Define APIError interface matching RFC 9457 Problem Details
   - Create `apiError(status, detail, title?)` helper function
   - Return `{ type: 'about:blank', title, status, detail }` format

2. **Optional Bearer auth middleware:**
   - Check for user context first (internal plugin calls skip auth)
   - If no user context, check for REDIRECTS_API_KEY env var
   - In production without API key, return 401
   - In dev without API key, allow access

3. **GET / (list with filtering):**
   - Parse query params: isActive (explicit string comparison), statusCode, matchType, search, limit, offset
   - Call `service.list(filter)` and `service.count(filter)` in parallel
   - Return `{ data: redirects, pagination: { limit, offset, total } }`

4. **GET /:id (read single):**
   - Call `service.getById(id)`
   - Return 404 with apiError if not found
   - Return `{ data: redirect }` if found

5. **POST / (create):**
   - Parse JSON body as CreateRedirectInput
   - Basic validation: source and destination required
   - Get userId from `c.get('user')?.id || 'api'`
   - Call `service.create(body, userId)`
   - Return 400 with apiError if service returns error
   - Return 201 with `{ data: result.redirect }` on success

6. **PUT /:id (update):**
   - Parse JSON body as UpdateRedirectInput
   - Call `service.update(id, body)`
   - Return 404 if 'Redirect not found', 400 for other errors
   - Return `{ data: result.redirect }` on success

7. **DELETE /:id (delete):**
   - Call `service.delete(id)`
   - Return 404 if 'Redirect not found', 400 for other errors
   - Return `{ success: true }` on success

8. **Database access pattern:**
   - Use `c.env?.DB || c.get('db')` like admin routes
   - Return 503 with apiError if DB unavailable

9. **Error handling:**
   - Wrap all handlers in try/catch
   - Log errors with console.error
   - Return 500 with generic apiError message

Export `createRedirectApiRoutes` function that returns the Hono router.

Update `src/plugins/redirect-management/index.ts` to mount API routes:

1. **Add import:**
   ```typescript

   ```

2. **Add export for direct mounting (like admin routes):**
   ```typescript

   ```

3. **Mount API routes in plugin builder:**
   After the existing admin routes mount, add:
   ```typescript
   builder.addRoute('/api/redirects', createRedirectApiRoutes(), {
     description: 'Redirect management REST API',
     requiresAuth: false,  // API handles its own auth via Bearer tokens
     priority: 100
   })
   ```

Then verify the endpoints work:

1. Start the dev server if not running: `npm run dev`

2. Test list endpoint:
   ```bash
   curl -s http://localhost:3000/api/redirects | jq .
   ```
   Should return `{ data: [...], pagination: { limit, offset, total } }`

3. Test create endpoint:
   ```bash
   curl -s -X POST http://localhost:3000/api/redirects \
     -H "Content-Type: application/json" \
     -d '{"source": "/api-test", "destination": "https://example.com"}' | jq .
   ```
   Should return 201 with `{ data: { id: "...", source: "/api-test", ... } }`

4. Test read endpoint (use ID from create response):
   ```bash
   curl -s http://localhost:3000/api/redirects/{ID} | jq .
   ```
   Should return `{ data: { ... } }`

5. Test update endpoint:
   ```bash
   curl -s -X PUT http://localhost:3000/api/redirects/{ID} \
     -H "Content-Type: application/json" \
     -d '{"destination": "https://example.com/updated"}' | jq .
   ```
   Should return `{ data: { ... destination: "https://example.com/updated" ... } }`

6. Test delete endpoint:
   ```bash
   curl -s -X DELETE http://localhost:3000/api/redirects/{ID} | jq .
   ```
   Should return `{ success: true }`

7. Test 404 case:
   ```bash
   curl -s http://localhost:3000/api/redirects/nonexistent-id | jq .
   ```
   Should return 404 with `{ type: "about:blank", title: "Not Found", status: 404, detail: "..." }`

8. Test filter params:
   ```bash
   curl -s "http://localhost:3000/api/redirects?isActive=true&limit=5" | jq .
   ```
   Should return filtered results with pagination.total reflecting filter.
